## 3. Core Domain Models

### 3.1 TranslatedPayload (THE central contract)

```go
// internal/config/types.go

package config

import "time"

// TranslatedPayload is the canonical data format for all data flowing through BubbleFish.
// Every component — WAL, queue, destination, presenter — depends on this struct.
// Changes to this struct REQUIRE a schema_version bump and a migration function.
type TranslatedPayload struct {
    ID             string                 `json:"id"`                        // UUIDv4, generated by BubbleFish
    Timestamp      time.Time              `json:"timestamp"`                 // UTC, set on receipt
    Source         string                 `json:"source"`                    // Source name from TOML config
    Content        string                 `json:"content"`                   // Primary content field
    Metadata       map[string]interface{} `json:"metadata"`                  // Arbitrary key-value pairs
    Vector         []float32              `json:"vector,omitempty"`          // Embedding vector (required for ChromaDB)
    SchemaVersion  string                 `json:"schema_version"`            // e.g., "1.0.0"
    IdempotencyKey string                 `json:"idempotency_key,omitempty"` // Optional dedup key
}

const CurrentSchemaVersion = "1.0.0"
```

```go
type QueryRequest struct {
```

### 3.3 FieldMapping

```go

type FieldMapping struct {
```

### 3.4 TransformRule

```go
// internal/transform/engine.go

type TransformRule struct {
    TargetField string `json:"target_field"` // Field in TranslatedPayload to write to

}
```

### 3.5 Error Types

```go
// internal/config/errors.go

type ErrorClass int

const (
    ErrorTransient   ErrorClass = iota // Retry with exponential backoff (max 3 attempts)
    ErrorPermanent                      // Drop payload, log error, do not retry
    ErrorSchema                         // Build-time error — invalid config, fail build
    ErrorAuth                           // Auth failure — reject immediately with 401/403
)

type ClassifiedError struct {
    Class   ErrorClass
    Message string
    Source  string // Which source triggered the error
    Err     error  // Underlying error
}

func (e *ClassifiedError) Error() string {
    return fmt.Sprintf("[%s] %s: %s", e.Source, e.Class, e.Message)
}
```

## 4. Core Interfaces

### 4.1 Destination Plugin

```go
// internal/destination/plugin.go

package destination

import (
    "context"
    "github.com/shawnsammartano/bubblefish/internal/config"
)

// Destination is the interface that all destination plugins must implement.
// Each destination gets its own queue worker(s) and circuit breaker.
type Destination interface {
    // Name returns the destination identifier (matches TOML filename without extension)
    Name() string

    // Init establishes connection to the destination. Called once at daemon startup.
    Init(ctx context.Context, cfg config.DestinationConfig) error

    // Write persists a single payload. Called by queue worker goroutine.
    // Return a ClassifiedError so the worker knows whether to retry.
    Write(ctx context.Context, payload config.TranslatedPayload) error

    // Read queries the destination. Called by GET /query/{dest} handler.
    Read(ctx context.Context, query config.QueryRequest) ([]map[string]interface{}, error)

    // HealthCheck verifies the destination is reachable and writable.
    // Called periodically by the circuit breaker. Return nil if healthy.
    HealthCheck(ctx context.Context) error

    // Close releases resources. Called during graceful shutdown.
    Close() error
}
```

### 4.2 WAL

```go
// internal/wal/wal.go

package wal

import "github.com/shawnsammartano/bubblefish/internal/config"

// WAL is the write-ahead log interface.
// Community: mutex-protected JSONL with size-based auto-rotation.
// Enterprise: extends with ring buffer, group commit, encryption.
type WAL interface {
    // Append writes a payload to the WAL. MUST complete before queue enqueue.
    // Includes schema_version in the entry for forward-compatible replay.
    Append(payload config.TranslatedPayload) error

    // Replay reads WAL entries from the given offset forward.
    // Used on startup to recover payloads that were logged but not yet written.
    // If an entry's schema_version != current, apply migration or log warning.
    Replay(fromOffset int64) ([]config.TranslatedPayload, error)

    // Rotate forces a segment rotation. Auto-rotation also happens at max segment size.
    Rotate() error

    // LastOffset returns the current write offset for crash recovery tracking.
    LastOffset() int64

    // Close fsyncs and closes the WAL file. Called during graceful shutdown.
    Close() error
}
```

### 4.3 Queue

```go
// internal/queue/queue.go

package queue

import (
    "context"
    "time"
    "github.com/shawnsammartano/bubblefish/internal/config"
)

type QueueStats struct {
    Waiting    int    `json:"waiting"`
    Processing int    `json:"processing"`
    Completed  int64  `json:"completed"`
    Errors     int64  `json:"errors"`
    Destination string `json:"destination"`
}

// Queue manages the buffered channel and worker goroutine(s) for a single destination.
type Queue interface {
    // Enqueue adds a payload to the queue. Returns error if load shedding threshold reached.
    Enqueue(payload config.TranslatedPayload) error

    // StartWorkers launches worker goroutines that consume from the queue.
    StartWorkers(ctx context.Context)

    // Stats returns current queue metrics for the API and dashboards.
    Stats() QueueStats

    // Drain blocks until all in-flight items are processed or timeout is reached.
    // Called during graceful shutdown.
    Drain(timeout time.Duration) error
}
```

### 4.4 IdempotencyStore

```go
// internal/idempotency/store.go

package idempotency

// IdempotencyStore tracks recently-seen idempotency keys to prevent duplicate writes.

type IdempotencyStore interface {
```

## 6. HTTP API Specification

### 6.1 POST /inbound/{source_endpoint}

**Request:**
```
POST /inbound/claude
Authorization: Bearer <source_api_key>
Content-Type: application/json
Content-Length: <must be <= max_payload_bytes>

{
  "message": { "content": "Hello world" },
  "model": "claude-3-opus",
  "metadata": { "user_id": "abc123" }
}
```

**Success:** `200 OK` — `{"status": "accepted", "id": "<uuid>"}`

**Errors:**
- `401 Unauthorized` — Invalid or missing API key
- `404 Not Found` — Unknown source endpoint
- `413 Payload Too Large` — Body exceeds max_payload_bytes
- `429 Too Many Requests` — Rate limit or load shedding. Includes `Retry-After: <seconds>` header.
- `503 Service Unavailable` — WAL disk full or daemon shutting down

### 6.2 GET /query/{destination}

**Request:**
```
GET /query/sqlite?limit=10&offset=0
Authorization: Bearer <source_api_key>
```

**Success:** `200 OK` — JSON array of filtered results

**Errors:**
- `401 Unauthorized` — Invalid API key
- `403 Forbidden` — Source does not have can_read=true for this destination
- `503 Service Unavailable` — Destination circuit breaker is open

### 6.3 GET /api/status

**Request:**
```
GET /api/status
Authorization: Bearer <api_token from daemon.toml>
```

**Response:**
```json
{
  "version": "1.0.0",
  "uptime_seconds": 3600,
```

## 8. WAL Replay Behavior on Schema Version Mismatch

When Replay() encounters a WAL entry with a schema_version older than CurrentSchemaVersion:

1. Look up a registered migration function: `migrations[entry.SchemaVersion]`
2. If a migration exists, apply it to transform the entry to CurrentSchemaVersion. Log: `"Migrated WAL entry <id> from schema v<old> to v<current>"`
3. If NO migration exists for that version gap, **log a warning and skip the entry**: `"WARN: Skipping WAL entry <id> — no migration from schema v<old> to v<current>. Entry preserved in WAL segment for manual recovery."`
4. The entry is NOT deleted from the WAL. It remains in the segment file for manual inspection.

Migration functions are registered at daemon startup:
```go
// internal/schema/version.go
type MigrationFunc func(old TranslatedPayload) (TranslatedPayload, error)

var migrations = map[string]MigrationFunc{
    // "0.9.0": migrateV09ToV10,
}
```

## 9. ChromaDB Embedding Clarification

BubbleFish does NOT generate embeddings. BubbleFish routes data. If a destination requires vectors (ChromaDB with require_vector=true), the inbound payload MUST include an embedding vector. This is enforced at two levels:

1. **Build time:** If a source targets a ChromaDB destination with require_vector=true, the build compiler checks that the source mapping includes a field mapping to the `vector` field. If missing, build fails with: `"ERROR: Source 'claude' targets ChromaDB destination 'vectors' which requires vectors, but no mapping to 'vector' field is defined."`

2. **Runtime:** If a payload arrives without a vector field (or with an empty vector), the daemon returns 400 Bad Request: `{"error": "ChromaDB destination requires embedding vector. Include vector field in payload or use an AI client that generates embeddings."}`
